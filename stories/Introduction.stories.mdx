# Tiger21's React Native UI Components

- This is hosted version of a [**storybook**](https://storybook.js.org/). Its code can be found [**on github here**](https://github.com/TIGER21-LLC/react-native-ui-components).
- Storybook helps developers code reusable UI components in isolation from an app's business logic, context, and API backend.
  - Not exactly, but you can think of this as pre-fabricated construction of a house's walls, windows, furniture, etcetera. These are not models, they are the actual stuff, that will be later on assembled on-site to build the whole thing.
  - In the real world houses constructed with pre-fabricated components is probably less desirable. But in the digital world of react-native it leads to better quality product, built faster, more maintainable, testable, etcetera, and is more desirable for the reasons listed below.
- Storybook also creates a browseable catalog of all the UI components.
  - This makes it easy to develop hard-to-reach states of an app.
  - Automated visual tests can run on each UI component separately and their collection together.
  - Humans can browse every UI component (aka "story") by navigating to them in the sidebar.
- Storybook recommends building UIs with a [**component-driven**](https://componentdriven.org) process starting with atomic components and ending with pages.
